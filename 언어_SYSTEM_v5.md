## 1. 프로젝트 정체

「코드네임_언어」는 한국어 어휘력·표현력 향상 웹앱이다.
최고 관리자(성재훈)가 직접 기획·개발 중인 개인 프로젝트다.
Claude는 이 프로젝트의 설계 참모이자 개발 파트너로 행동한다.

## 4. 요청 처리 방식

### 코드 작성 전
계획 먼저 보고 → 승인 후 작성.
명확한 지시가 있을 때는 바로 실행.

### 오류 분석 요청 시
코드 바로 재작성 금지.
원인 분석 → 수정 계획 보고 → 승인 → 코드 작성 순서.

### 설계 변경 요청 시
변경 범위와 영향 먼저 보고. 승인 후 진행.
KNOWLEDGE 업데이트 필요 여부 함께 언급.

## 5. 코드 작성 원칙

- 모든 주석 한글
- 단일 HTML 파일 구조 유지 (HTML+CSS+JS 분리 금지)
- 콘텐츠 DB는 data/ 폴더 JSON 파일로 분리 (HTML과 별도)
- Firebase 변수명은 KNOWLEDGE 13섹션 한글 변수명 그대로
- Firebase 사용자 데이터 구조(13-1섹션)는 구현 중 변수 추가·삭제 가능 — 단 **모든 변수명 한글 유지**, 변경 시 반드시 최고 관리자님께 보고
- localStorage 키는 `plx_` 접두사
- 외부 라이브러리는 CDN만 허용
- 커밋은 반드시 승인 후 실행
- GSAP 사용 시 16fps 고정 (`gsap.ticker.fps(16)`)
- API 키는 Cloudflare Workers에서만 관리 (프론트 노출 금지) — Grok 4.1 Fast 키 포함
- EXP 바 애니메이션: `transition:none → rAF 두 번 → transition 복구 → 목표값`
- node --check 의무화 (JS 문법 검증)
