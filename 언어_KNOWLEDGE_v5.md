## 2-2. 학습 화면 설정 패널

각 학습 화면 상단에 접이식 설정 패널 배치.
기본적으로 접혀있고 ⚙ 탭하면 펼쳐짐. 마지막 설정값 저장됨.

```
┌─────────────────────────────┐
│ ⚙ 학습 설정      [접기 ▲] │
└─────────────────────────────┘
```

모드별 설정 옵션 (v3.7 갱신):

| 모드 | 설정 옵션 |
|---|---|
| 플래시카드 (고사성어·속담 / 한자·우리말) | 방식: 4지선다 / 뜻 직접 서술 / 역방향 / 유의어 변별 |
| 4지선다 (상식·어원 / 세계사·신화) | 방식: 선택지 / 직접입력 |
| 3지선다 (맞춤법) | 방식: 선택지 / 직접입력 |
| 아재개그 | 난이도: 아↗그거! / UP¡¿ + 입력 방식: 선택지 / 직접입력 |
| 구어 교정 | 없음 (텍스트 기본, 음성 입력 차단 — KNOWLEDGE 6-1) |

**학습 방식 5가지 상세 (KNOWLEDGE 3-1):**
- 4지선다: **기본값.** 선지는 뉘앙스 유사하게 설계
- 뜻 직접 서술: 자신의 말로 써보고 실제 뜻과 비교 (AI 채점 없음)
- 역방향: 뜻 보여주고 단어 맞추기
- 유의어 변별: 예문 빈칸에 유의어 중 선택 → correct·acceptable·wrong 3단계 색 분기 (KNOWLEDGE 3-2)

**기타 방식 상세:**
- 직접입력: 선택지 없이 직접 입력 → 정답 비교
- 입력검증: (v3.6 이전) 뜻을 직접 타이핑 → AI 정답 여부 판정 후 공개 — 「뜻 직접 서술」로 흡수됨

**아재개그 입력 방식 (v3.7 신규):**
- 선택지: 탭→공개 (기존 방식)
- 직접입력: 정답을 직접 타이핑하여 비교

**상식·어원** — 비문학 상식 (악수 기원, 잡학 등) + 영어 어원 포함. 4지선다.

**고사성어·속담** — 직설적 톤. 4지선다 기본, 학습 설정에서 5방식 전환 가능. 뜻+한자+직설+예문+연상법 레이어.

**세계사·신화** — 교과서 세계사 아님. 고대 로마 신화·세이렌 등 나무위키풍 병맛 방향.

**아재개그** — 탭하면 정답+뜻(설명) 공개. 선택지 없음.

난이도 2단계:
- `아↗그거!` — 누구나 한 번쯤 들어봤을 것 (예: 왕이 넘어지면? → 킹콩)
- `UP¡¿` — 엥? 이게 뭔데. 지식+언어유희 혼합. AI 출제 + 최고 관리자님 직접 개입
  (예: 뚱뚱한 구름은? → 적운형 구름 / 함께 달에 가지 못해서 미안해 → 데이비드)

**한자·우리말** — 한자어: 한자 표기 함께 표시. 한자 탭하면 음+훈 공개. 순우리말: 🇰🇷 뱃지만 표시, 한자 없음.
```
예시 — 한자어:
  필연 (必然)
  탭 → 必 반드시 필 / 然 그럴 연

예시 — 순우리말:
  윤슬 🇰🇷
  (한자 탭 없음, 뜻+예문만)
```

**맞춤법** — 한국인이 자주 틀리는 맞춤법. 3지선다.

**구어 교정** — 텍스트 직접 입력만 허용. 음성 입력은 차단됨(v3.7 항목2: Permissions-Policy + insertFromSpeech/insertFromDictation 차단, KNOWLEDGE 6-1 참조). 일상 말투→격식 말투 변환 훈련. 짧은 발화 1~3문장. Grok이 유도리 있는 톤으로 교정.
```
예시:
  "고객님 환불은 불가능합니다"
  → "고객님, 환불이 어려운 점 양해 부탁드립니다"
```

## 3. 학습 화면 구조 (확정)

고사성어·속담 / 한자·우리말의 학습 방식은 학습 설정 **5가지** (3-1 참조, 기본값: **4지선다**).
단어 정보는 아래 ①~⑥ 데이터로 구성되며, ①읽기 ②뜻만 즉시 공개되고 ③~⑥은 「더 알아보기」 토글로 펼쳐 본다.

```
앞면: 苛斂誅求

탭 →

① 읽기    가렴주구              ← 즉시 공개
② 뜻      세금을 혹독하게...    ← 즉시 공개

[ 더 알아보기 ▾ ]               ← 탭하면 펼쳐짐

③ 한자    苛 가혹할 가 / 斂 거둘 렴...  (큰 폰트)
④ 직설    💬 "나랏님이 백성 등골 빼먹는 것"  (말풍선)
⑤ 예문    ❝ 탐관오리들의 가렴주구에... ❞   (인용 스타일)
⑥ 연상법  💡 가혹하게 렴치없이 주구장창...

─────────────────────────────
[알았다 ✓]  [헷갈린다 △]  [몰랐다 ✗]
```

### 3-1. 학습 방식 (학습 설정에서 선택, 5가지 확정)

| # | 방식 | 설명 |
|---|---|---|
| 1 | 4지선다 | 가장 정확한 뜻 고르기. **기본값.** 선지는 뉘앙스 유사한 것으로 난이도 설계 |
| 2 | 뜻 직접 서술 | 단어 보고 뜻을 자신의 말로 서술 → 실제 뜻과 나란히 비교 (AI 채점 없음) |
| 3 | 역방향 | 뜻 보여주고 단어 맞추기 |
| 4 | 유의어 변별 | 예문 빈칸에 유의어 중 가장 맞는 단어 선택. 선택 후 전체 유의어 뜻 공개 |
| 5 | 플래시카드 | 카드 앞면(단어) → 탭하면 뒷면(뜻) 공개. 전통 암기 방식 |

### 3-2. 유의어 변별 데이터 구조 (3단계)

```
correct    → 정답 (초록 강조)
acceptable → 근사 정답 (이 맥락에선 덜 정확하지만 틀리진 않음)
wrong      → 오답 (빨강 강조)
```

- 기본 출제 원칙: 예문을 명확한 맥락으로 설계해 복수 정답 원천 차단 (A안)
- 애매한 경우: 기존 「이의있음!」기능을 유의어 변별 맥락까지 확장하여 AI 판정 (C안)
- 실사용자 댓글 문체 예문 데이터 풀 추가 예정 (최고 관리자님 수집 데이터)

### 3-3. 예문 데이터 파일 (정령왕_통합_v2.json)

**파일 위치:** `data/정령왕_통합_v2.json`

**구성 (총 52건):**

| 분류 | 건수 | 비고 |
|---|---|---|

| 오늘의_한문장 | 7건 | 웹툰(정령왕 엘퀴네스) 대사만, 댓글 없음 |

## 4. AI 출제 시스템

### 기본 비율
- **DB 40% + AI 즉석 생성 60%**
- AI 생성 문제에 🤖 뱃지 표시
- **Grok 4.1 Fast 단일 API (Cloudflare Workers 관리)**
- 설정에서 「AI 문제만 출제」토글 가능

### 모드 진입 시 출제 흐름

```
모드 진입
│
├─ 확률 결정 (DB 40% / AI 60%)
│
├─ DB 선택 시
│   ├─ 히스토리 필터 적용 후 풀 문제 존재 → DB에서 출제
│   └─ 풀 문제 없음 (히스토리 제한으로 소진)
│       └─ 소형 팝업: "출제할 DB 문제가 없어 AI로 대체합니다"
│           → 토큰 소모 후 AI 출제로 자동 전환
│
└─ AI 선택 시
    └─ 토큰 소모 → Grok 즉석 생성 → 문제 표시
```

> - DB 소진 시 팝업은 화면 하단 토스트, 2초 표시 후 자동 소멸
> - DB 소진 강제 전환 시에도 토큰 차감 (15토큰) 동일 적용
> - 「AI 문제만 출제」ON 시 위 흐름 생략, 항상 AI 출제

### AI 출제 상세
- **문제 본문 + 선택지 + 정답 + 10자 이내 핵심 요약(히스토리용) + reasoning_note 동시 생성**
- DB 문제: GitHub 저장소 data/ JSON 파일에서 로드

## 5. 질문하기 & 이의있음! 기능

### 질문하기
- 학습 화면 하단에 상시 표시 (미니 입력창 형태)
- 탭하면 슬라이드업으로 전체 패널 등장
- **토큰 차감: 20토큰 시작, 4턴~ 매 턴 +15 (32섹션 토큰 시스템 참조)**
- **기존 하루 100회 공유 풀: 폐기 — 토큰 시스템으로 완전 통합**
- Firebase에 사용자별 호출 횟수 + 타임스탬프 저장
- [창조주] 달성 흐름 진입점

### 이의있음!

#### 흐름 (멀티턴 확정)
```
기존: 즉시 판정 (1회성)
변경: 멀티턴 대화 후 최종 판정

흐름:
1. 이의있음! 탭 (토큰 70 차감)
2. reasoning_note 패널 상단 표시 (33섹션 참조)
3. 사용자 반박 입력 (1턴)
4. Grok 4.1 Fast 리즈닝 high 1차 응답
5. 사용자 재반박 가능 (매 턴 +30토큰 차감)
6. 최종 판정 (인정 / 기각)
   - 인정 시: [반박의 화신] 카운터 +1
```

#### 표시 조건
- 질문하기 답변 생성 후에만 하단에 작은 버튼 노출
- 구어 교정 모드에서는 필터링 강화
- **유의어 변별 모드 확장**: "내가 고른 게 맞는 것 같은데?" 상황에서도 이의 제기 가능 → AI가 맥락 기반 판정
- **토큰 소진 시 비활성화**
- **턴 제한 없음 (토큰으로 자연 억제)**

#### 악용 방지
- 다른 채팅방에서 분당 3회 이상: `"?? 위협 감지??"` 10분 락
- 한 채팅에서 30회 이상 연속 5분 이내: 동일 락
- **1분간 10개 이상 입력: 동일 락** (신규)

#### rate limit 문구
```
"Oops, 그록이 뻗었어요!"
```
> ⚠️ rate limit 실제 수치(RPM/TPM)는 xAI Console > Rate Limits에서 확인 후 반영 필요 (보류 — 21섹션 참조)

## 6. 음성 인식

- Whisper.js (크로스 브라우저, 완전 무료, 서버리스)
- 크롬: WebGPU 가속 / 파이어폭스·사파리: WebAssembly(CPU)
- 첫 실행 시 40MB 모델 다운로드, 이후 캐시
- 구어 교정 모드 음성 선택 시 사용 (텍스트 직접 입력도 선택 가능)

### 6-1. 텍스트 탭 음성 입력 차단 (확정)

구어 교정 텍스트 탭에서 키보드 음성 입력 차단. 2가지 레이어 조합:

| 레이어 | 방법 | 차단 대상 |
|---|---|---|
| 1 | `Permissions-Policy: microphone=()` 메타 태그 | 지보드·구글 음성 |
| 2 | `beforeinput` 이벤트 `insertFromSpeech` + `insertFromDictation` 차단 | Android + iOS |

- 붙여넣기는 `insertFromPaste` inputType으로 구분 → 오분류 없음
- 삼성 키보드 온디바이스 음성 일부 뚫릴 수 있음 (OS 레벨 처리, 현실적 한계)

## 13-1. Firebase 사용자 데이터 구조

Firebase Firestore에 저장되는 전체 구조. 데모에서는 JS 더미 변수로 대체되며 Firebase 연동 시 이 구조로 교체.

> ⚠️ 이 섹션은 Claude가 추정 작성한 구조이며 Claude Code 단계에서 실제 구현 시 능동적으로 확정한다.
>
> **Claude Code 작업 원칙 (이 섹션 전용):**
> - 구현 중 필요한 변수는 자유롭게 추가 가능
> - 불필요하거나 중복된 변수는 삭제 가능
> - 단, **모든 변수명은 한글**로 유지
> - 변경 후 반드시 어디를 어떻게 바꾸었는지 최고 관리자님께 보고

---

### 루트 문서 (users/{uid})

루트에는 빠른 읽기가 필요한 값만 저장. 큰 배열·컬렉션은 서브컬렉션으로 분리.

**기본 사용자 정보**
```
이름          — 사용자가 설정한 닉네임 (string)
이메일        — 구글 계정 이메일 (string)
프로필이미지  — 이모지 또는 업로드 이미지 URL (string)
가입일        — 최초 로그인 일시 (timestamp)
```

**성장 데이터**
```
레벨          — 현재 레벨 1~70 (number) ※ 개발자모드 확장값은 오버레이에만 존재, DB 미저장
현재EXP       — 현재 레벨 내 누적 EXP (number)
총누적EXP     — 전체 누적 EXP. Lv.70 이후에도 계속 누적 (number)
연속학습일    — 오늘 포함 연속 학습일 수 (number)
총학습일      — 누적 학습일 수 (number)
마지막학습일  — 연속 학습일 계산 기준 (string, YYYY-MM-DD)
```

**마스터리 변수 (13섹션 참조)**
```
상식어원학습수 / 언어의뿌리학습수 / 세계사신화학습수
아재개그학습수 / 맞춤법학습수 / 구어교정횟수

```
※ 변수명 분리·추가는 13섹션 [검토 필요] 확정 후 반영

**토큰 데이터**
```
보유토큰          — 현재 잔여 토큰 (number, 기본 1500)
토큰소진시각      — 소진 타임스탬프 (timestamp | null)
토큰락해제시각    — 소진시각 + 2시간 (timestamp | null)
```

**설정 데이터**


**업적 데이터**
```

  단발 업적:  { "첫걸음": 0 또는 1 }  (0=미달성 ☽, 1=달성 ☾)
```

### 개발자 모드 기능 (오버레이 구조)

개발자 모드는 실제 계정 DB 위에 올라탄 오버레이 레이어로 동작.

실제 계정 DB는 개발자 모드 진입 여부와 무관하게 항상 실시간 업데이트됨 (화면에 보이지 않을 뿐).
강제 조작값은 오버레이 레이어에만 표시됨.
화면에 보이는 값 = 실제 계정 + 오버레이 조작값 합산.
원복 = 오버레이 레이어만 제거. 실제 계정 DB는 그동안 쌓인 그대로 유지.

예시:
- 개발자 모드에서 업적 전부 풀돌 강제 달성 → 원복 시 사라짐
- 개발자 모드 안에서 실제로 반박 성공 → [반박의 화신] 1돌이 실제 계정에 반영됨 → 원복해도 유지

**레벨 확장:** Lv.71~120까지 실제 진행 가능 (+50 해금)

**제한 해제:**
- 복습 대기열 상한 제한 해제 (무제한)
- 히스토리 필터 상한 제한 해제 (무제한)
- **토큰 시스템 제한 해제 (무제한 토큰)**
- 커스텀 테마 슬롯 제한 해제
- 고급 커스터마이징 Lv.16 조건 해제

**수정 가능 (전부 오버레이 내 동작):**
- EXP 직접 수정
- 레벨 직접 수정
- 마스터리 카운터 직접 수정
- 업적 명함~풀돌 강제 달성/해제
- 복습 대기열 항목 직접 추가/삭제

## 24. 오늘의 한 문장 시스템 (개편 확정)

### 위치
홈 화면 상단 슬라이드 배너

### 동작 방식
- CSS 자동 슬라이드 순환
- 명언 또는 문학적 문장을 슬라이드로 표시
- 문장 안에 학습 단어(필연·운명·숙명 등)가 자연스럽게 녹아있는 구조
- 문장 내 해당 단어 탭 → 해당 단어 플래시카드로 즉시 이동

### 콘텐츠 방식
- 하이브리드: DB 저장 문장 + AI 생성 문장 혼합
- 단어의 관계를 분석적으로 설명하는 방식 아님
- 문장을 통해 단어의 쓰임과 감각을 자연스럽게 체득하는 방식

### 예시
```
"운명의 별이 이어준 우리의 우연 같은 필연"
        ↑                              ↑
      탭 → 운명 플래시카드        탭 → 필연 플래시카드
```

### 데이터 구조
```
data/정령왕_통합_v2.json 의 오늘의_한문장 배열
→ 홈 진입 시 JSON fetch → 슬라이드로 표시
```

## 33. reasoning_note 시스템

### 생성
- **AI 출제(Grok) 문제 생성 시 동시 생성** — 문제 본문·선택지·정답·히스토리요약과 함께 1회 호출로 처리
- DB 문제: 최고 관리자님이 JSON에 직접 `reasoning_note` 필드 작성 (확정)
  - **fallback**: `reasoning_note` 필드가 없으면 Grok이 DB 문제 내용 기반으로 즉석 생성
  - ⚠️ 주석: JSON 작업 시 반드시 `reasoning_note` 필드 확인할 것. 없으면 자동으로 Grok fallback 실행됨

### 저장 구조 (Firebase)
```
users/{uid}/history/{모드}/{문제id}
  → title:          "괴로울 vs 힘들"         (히스토리 요약, 10자 이내)
  → date:           timestamp
  → reasoning_note: "출제 의도 및 판단 근거..."
  → result:         "정답/오답"
```

### 표시 위치
- **이의있음! 패널 상단** (항상 표시, 5섹션 참조)
- **히스토리 화면** (아코디언, 펼치면 표시)

학습 탭 최상단

- 최근 5개 차감 내역 (Firebase 토큰로그 서브컬렉션 실시간 로드)
- 형식: 기능명 | 차감량 | 잔여 | 시간

### 차감 안내 팝업 (버튼 탭 시)
토큰 바 하단에 작은 버튼 [? 차감 안내] 배치.

```

├──────────────────────────┤
│  문제 생성      -15       │
│  질문하기       -20 (시작)│
│               +15 / 4턴~│
│  사고전개      -100 (시작)│
│               +30 / 4턴~│
│  구어 교정      -30       │
│  유사 문제      -20       │
│  이의있음!      -70 (시작)│
```
